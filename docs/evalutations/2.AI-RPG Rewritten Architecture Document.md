**# AI-RPG Rewritten Architecture Document**

**Version**: 3.0 (Incorporating Aventuras & Talemate Inspirations)  
**Status**: Recommended Evolution  
**Core Philosophy Retained**: Rules-agnostic "Lego Protocol" using declarative System Manifests and Prefabs. The LLM acts as a flexible reasoning brain that proposes actions via a minimal toolset. Deterministic code (Validation Pipeline + DeterministicCruncher + WorldStateGuardian) enforces mechanical rules, narrative truths, and consistency. State-first design with structured outputs remains central.

This version integrates high-value patterns from **Aventuras** (rich Lorebook with autonomous management, chapter-based summarization with rich metadata, in-story time tracking, named checkpoints with retry/branching, collapsible card inspectors, and image generation hooks) and **Talemate** (light multi-agent specialization, dedicated world-state consistency engine, Jinja2-style dynamic prompt templating, and node-based authoring for scenarios/prefabs). These additions enhance long-term coherence, authorability, and player immersion while preserving performance focus, minimal tool complexity, and deterministic offloading.

### Executive Summary of Changes (v2.0 → v3.0)

- **Memory & Lore**: HierarchicalMemoryRetriever now includes Aventuras-style chapter segmentation, rich metadata (keywords, plot threads, in-story time, entities), and a dedicated Lorebook module with autonomous background Lore Agent for dynamic state tracking and relevance injection.
- **Consistency & Agents**: New WorldStateGuardian (inspired by Talemate’s world-state engine) works alongside DeterministicCruncher for both mechanical and narrative truth enforcement. Light multi-agent coordination via ModelRouter (Narrator, Lore Agent, Guardian roles).
- **Context & Prompting**: ContextBuilder upgraded with Jinja2-style templating for flexible, conditional Manifest/Lorebook injection.
- **Persistence & UX**: StateSnapshotManager enhanced with named checkpoints, state-preserving retry, and branching (Aventuras). UI gains collapsible card inspectors, node editor for Prefabs/quests, and image generation hooks.
- **Turn Flow**: Short ReAct loop now includes targeted lore injection and lightweight consistency reflection. In-story time tracking becomes first-class.
- **Performance**: All new background agents and summarization are throttled via TokenBudgetManager to maintain latency/cost gains.

The architecture stays layered, maintainable, and extensible.

### Updated Toolset

The minimal atomic set is preserved for low decision complexity. Enhancements focus on better Lorebook integration and Manifest-aware schemas.

**LLM-facing Tools** (snake_case, strict JSON Schema with dynamic descriptions):
1. `adjust_attribute` – Numeric/derived stat changes.
2. `set_property` – Non-numeric flags, strings, or complex values.
3. `roll_dice` – Dice rolls with modifiers (routed to DeterministicCruncher).
4. `mark_flag` – Binary/enumerated conditions or quest updates (feeds Lorebook).
5. `move_entity` – Relocation of characters/items.
6. `add_note` – Narrative/lore entries (directly augments Lorebook with metadata support).

Tool descriptions are dynamically templated with relevant Manifest excerpts, active Lorebook entries, and examples via Jinja2.

### Revised Core Flows

#### Turn Execution Flow (Hybrid Planning + ReAct + Consistency)

```mermaid
sequenceDiagram
    participant User
    participant TM as HybridTurnManager
    participant CB as ContextBuilder
    participant MR as HierarchicalMemoryRetriever
    participant LB as Lorebook + Lore Agent
    participant LLM as LLM (routed)
    participant TE as ToolExecutor
    participant DC as DeterministicCruncher
    participant WSG as WorldStateGuardian

    User->>TM: User Input
    TM->>CB: Build Context (Jinja2 template: Manifest + Dynamic + Budgeted Memory + Relevant Lorebook)
    TM->>LB: Inject keyword/relevance-triggered lore
    TM->>LLM: Structured Planning Call

    loop Short ReAct (max 8, target 4-6)
        TM->>LLM: Tool-calling thought (fast model)
        alt Tool Calls
            TM->>TE: Execute Tools
            TE->>DC: Resolve dice/formulas + time progression
            TE->>WSG: Narrative consistency check
            TE-->>TM: Validated Results
            TM->>TM: Lightweight Reflection (Manifest + Lore truths)
        else Narrative Ready
            Break
        end
    end

    TM->>LLM: Narrative Streaming (premium model)
    TM->>LLM: Structured Suggestions
    TM->>LB: Background Lore Agent proposal (throttled)
    TM->>Storage: Persist Snapshot + Chapter Metadata + Lore
    TM->>UI: Emit Events (incl. IMAGE_GEN if triggered)
```

**Key Integrations**:
- Jinja2 templating for conditional context (e.g., “if quest active, inject these lore entries”).
- In-story time tracking advanced by DeterministicCruncher.
- WorldStateGuardian enforces narrative truths (e.g., dead characters stay dead unless rules allow).
- Autonomous Lore Agent runs post-turn in background for new/updated entries.

### New & Enhanced Components

- **Lorebook** (Aventuras-inspired): Unified entries for characters, locations, items, factions, events. Supports dynamic state (relationships, discoveries, hidden info), aliases, keyword/relevance injection, and import/export (JSON, YAML, SillyTavern cards). Autonomous background Lore Agent proposes updates reviewed by WorldStateGuardian.
- **WorldStateGuardian** (Talemate-inspired): Dedicated consistency layer that reinforces both mechanical invariants (via Validation Pipeline) and narrative truths (character facts, world continuity). Works with hierarchical memory.
- **HierarchicalMemoryRetriever**: Enhanced with chapter segmentation, rich metadata tagging (keywords, plot threads, in-story time, involved entities), automatic summarization at token thresholds, and AI-powered relevant event retrieval.
- **ContextBuilder**: Now uses Jinja2-style templating for highly dynamic, conditional prompt construction (Manifest, Lorebook, memory tiers).
- **StateSnapshotManager**: Named checkpoints with full state + lore snapshots. Supports state-preserving retry (re-run narrative with different seed/temperature) and branching (“what-if” paths).
- **ModelRouter**: Extended for per-agent routing (e.g., fast model for Lore Agent, premium for Narrator).
- **UI Enhancements**: Collapsible card-based inspectors (adaptive to Manifest and Lorebook), node editor for building complex Prefabs, quests, and branching scenarios (Talemate-style), and optional image generation hooks (Aventuras-style) triggered post-narrative.

### Observability & Reliability Additions

- Tracing now includes lore injection hits, chapter metadata quality, WorldStateGuardian interventions, and snapshot/branch history.
- Graceful degradation: Validation or consistency failures trigger reflection and clear user feedback.
- Throttling for background agents (Lore Agent, summarization) via TokenBudgetManager.
